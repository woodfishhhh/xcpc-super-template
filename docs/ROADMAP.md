# Roadmap

超级板子的目标是成为一个离线优先、开源、可比赛前直接使用的 ACM/XCPC 打印稿生成工具。路线图按可发布版本推进，每个阶段都必须保留可验证的质量门槛。

## 成品标准

- 公共模板库结构稳定，新增算法只需要添加 `meta.json` 和 `code.md`。
- 用户无需账号即可完成选模板、编辑、排序、配置介绍详细度、导出 Markdown/PDF。
- PDF 目录页码可信，目录上的页码对应实际标题所在页。
- 首次加载后可离线打开，支持个人模板编辑、导入导出和生成打印稿。
- 公共模板来源、许可证、复杂度和说明清晰。
- 仓库具备 CI、贡献规范、模板校验、E2E 主流程测试。

## v0.1 Stabilize Prototype

目标：把当前原型从“能跑”推进到“主流程稳定”。

- 固化模板库校验脚本。
- 建立 GitHub Actions：模板校验、单元测试、构建、E2E、high severity audit。
- 把 Playwright 主流程测试纳入仓库。
- 清理临时 QA 文件和无关产物。
- 修复首批真实使用反馈中的高优先级交互问题。

验收：

- `npm ci`
- `npm run validate:templates`
- `npm test`
- `npm run build`
- `npm run test:e2e`

## v0.2 Public Template Library

目标：让公共板子库成为项目核心资产。

- 固定 `板子/meta.schema.json`。
- 扩充到 30-50 个高频模板。
- 分类覆盖：基础算法、数据结构、图论、字符串、数学与数论、动态规划、几何、网络流、杂项。
- 每个模板保留来源和许可证。
- 校验重复 id、空代码、分类路径不匹配、缺字段。

验收：

- 所有公共模板通过 `npm run validate:templates`。
- 每个模板都能在应用内浏览、选择、编辑覆盖、恢复默认。

## v0.3 PDF Reliability

目标：让导出的 PDF 可以放心打印。

- 稳定目录页码生成。
- 优化页眉、页脚、页码和章节标题。
- 避免标题孤立在页底。
- 改善长代码、长复杂度、中文英文混排、超长模板标题。
- 增加分页后的 PDF 预览或可视化 QA 样本。

验收：

- 紧凑比赛版和书籍章节版都能生成可信页码。
- 大型打印稿不会出现明显横向截断或不可读代码。

## v0.4 Personal Library

目标：让用户敢把自己的队伍模板放进来。

- 个人模板文件夹/分类管理。
- 复制公共模板为个人模板。
- 导入冲突处理和错误提示。
- 单模板导入导出。
- 编辑器未保存状态和自动草稿。
- 搜索覆盖标题、分类、标签、复杂度和代码内容。

验收：

- 用户能维护自己的小型模板库。
- 坏 JSON 不会破坏已有个人库。

## v0.5 Draft Composition

目标：提升大规模打印稿编排效率。

- 章节级拖拽和重组。
- 学习顺序、字典序、分类顺序、自定义顺序。
- 批量详细度设置。
- 预设包：新生版、区域赛版、图论强化版、数学强化版。
- 生成前检查：空代码、重复模板、过长页面、无复杂度。

验收：

- 用户可以在 5 分钟内生成一份可打印稿。
- 大量模板时仍能快速整理。

## v0.6 Open Source Readiness

目标：让外部贡献者可以放心参与。

- 完善 `CONTRIBUTING.md`。
- 补全 issue template 和 PR template。
- 明确 GPL-3.0 下的模板来源要求。
- README 加截图、在线地址、部署方式。
- 发布贡献示例模板。

验收：

- 新贡献者能按文档提交一个公共模板。
- PR 中能自动跑完模板校验和测试。

## v0.7 Public Beta

目标：公开试用并收集真实反馈。

- 部署到稳定线上地址。
- 验证 PWA 离线缓存。
- 添加反馈入口。
- 找 3-5 个 ACM/OI 用户试用。
- 按真实反馈修复高频问题。

验收：

- 线上地址可访问。
- 断网后可打开并生成打印稿。
- 公开测试反馈被整理成 issues。

## v1.0 Release

目标：冻结核心契约并发布正式版。

- 冻结模板 schema。
- 冻结导出格式。
- 修完公开测试高优先级 bug。
- 整理 changelog。
- 打 `v1.0.0` tag。
- 写正式 release notes。

验收：

- CI 全绿。
- 核心 E2E 全绿。
- 线上版本可用。
- 公共模板库具备基础覆盖。
- PDF 打印稿可比赛使用。
